# Client Guide: Feedback & Bug Reporting

This guide outlines how to provide feedback and report bugs during and after the Mismo project build.

---

## 📝 Providing Feedback

Feedback is a critical part of the Mismo project lifecycle. We ask for your input at several stages:

### Stage 1: PRD Discovery
- Review the `PRD` generated by our agents.
- Comment on specific sections (e.g., API spec, data model).
- Mark sections as `resolved` once your feedback is addressed.

### Stage 2: Build Complete
- Once the build is successful, you'll receive a `FEEDBACK_REQUEST` notification.
- Rate your experience on a scale of 1-10.
- Provide optional comments on the quality of the code, documentation, and communication.

---

## 🐛 Bug Reporting

If you encounter an issue with your delivered project:

### During the Build
- Report any discrepancies in the `PRD` or `BuildLog`.
- Our engineers will review and address the issue within the current build cycle.

### Post-Delivery
- After the project is transferred to your GitHub repository:
1.  Navigate to your repository on GitHub.
2.  Go to the `Issues` tab.
3.  Click `New Issue`.
4.  Use the `Bug Report` template to describe the issue, provide reproduction steps, and include screenshots if possible.

**Bug Report Template:**
```markdown
### Summary
A clear and concise description of the bug.

### Reproduction Steps
1. Step 1
2. Step 2
3. Step 3

### Expected Behavior
What should have happened.

### Actual Behavior
What actually happened.

### Environment
- OS:
- Browser:
- Device:
```

---

## 📅 Maintenance Mode

If you've opted into the **Mismo Maintenance Plan**:

- Our agents will automatically check for dependency updates and security vulnerabilities every month.
- We'll create a `Pull Request` (PR) with any necessary patches for your review.
- You can provide feedback on these PRs directly through GitHub.
- If a critical security issue is identified, you'll receive an automated alert via email/Slack.
