# Mismo Infrastructure Setup - Fixes & Automation Log (2026-03-03)

This document summarizes the fixes and automation implemented to complete Phase 5, 6, and 7 of the Mismo distributed infrastructure setup.

## 1. Environment Variable Automation (Phase 5 & 6)

**Issue**: Manual setup of `.env` files across 4 machines was error-prone and inconsistent.
**Fix**: Centralized management on the MacBook Pro (Control Node) using Ansible.

- **Shared Secrets**: Generated secure, random secrets for `N8N_ENCRYPTION_KEY`, `POSTGRES_PASSWORD`, and `REDIS_PASSWORD`.
- **Ansible Playbook**: Created `mac-studios-iac/ansible/deploy-env.yml` to automate distribution.
- **Dynamic Templates**: Created `mac-studios-iac/ansible/templates/n8n-env.j2` to generate role-specific `.env` files:
  - **Studio 1**: Main node configuration with full database and monitoring access.
  - **Studio 2 & 3**: Worker configuration pointing to Studio 1's Tailscale IP.

## 2. n8n High-Availability Fixes

**Issue**: n8n-main was in a crash loop due to encryption key mismatch; workers couldn't connect.
**Fixes**:

- **Encryption Key**: Retrieved the existing key from Studio 1's Docker volume and synced it across all nodes.
- **Port Exposure**: Updated `docker-compose.main.yml` to expose Postgres (5432) and Redis (6379) to the host network.
- **Firewall**: Added a macOS `pf` anchor (`mismo-n8n`) on Studio 1 to allow inbound traffic from the Tailscale network (`100.64.0.0/10`).
- **Secure Cookies**: Set `N8N_SECURE_COOKIE=false` and updated `WEBHOOK_URL` to `http` to allow UI access over Tailscale without TLS.
- **Worker Cleanup**: Cleared stale n8n configuration volumes on Studio 2 and 3 to resolve secondary encryption mismatches.

## 3. Farm Monitor & Supabase Integration

**Issue**: Farm Monitor failing with SSH key errors and Supabase `403 Forbidden`.
**Fixes**:

- **SSH Key Distribution**: Automated the deployment of the private SSH key (`id_ed25519`) to Studio 1's `~/.ssh` directory with `0600` permissions for secure monitoring access.
- **Supabase Permissions**: Identified and resolved a schema-level permission issue. Granted `USAGE` on `public` schema and `ALL PRIVILEGES` to the `service_role`.
- **Database Migration**: Performed a manual SQL migration of the `ArchTemplate` enum values in Supabase to match the updated Prisma schema, followed by a successful `pnpm db:push`.

## 4. Final Cluster Status

- **Control Node (MBP)**: Single source of truth for configuration and orchestration.
- **Studio 1 (Control Plane)**: n8n-main, Postgres, Redis, and Farm Monitor all healthy and accessible.
- **Studio 2 & 3 (Execution Plane)**: n8n workers successfully connected to the queue and ready for builds.
- **Monitoring**: Active and reporting to Supabase.

**Access**: n8n UI is available at `http://100.124.80.81:5678`
